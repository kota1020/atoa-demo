# AtoA — AI社員オンボーディング＆ダッシュボード（ハッカソン デモUI）

「AI社員を会社に迎え入れる」体験のデモWebアプリ。バックエンドなし・全部ダミーデータの1ファイル構成。

## 起動

```bash
bash ~/ai-employee/serve.sh
# → http://localhost:8899
```

（`index.html` を直接開いてもOK。カメラ機能はlocalhost経由推奨）

## 体験フロー

1. **Hero** — 「AI社員を、あなたの会社へ。」
2. **迎え入れ** — 名前・配属先を決める
3. **オンボーディング研修** — ローカルフォルダ/コードのスキャン演出（ダミー）
4. **理解度診断** — 総合80%。「あなたの癖」だけ58%で足りない
5. **癖分析** — カメラ or 画面共有（本物の getUserMedia / getDisplayMedia）。「デモで進める」ボタンでカメラなしでも進行。理解度 80%→96%
6. **マイダッシュボード** — 社員ステータス / A2A商談（LINEで会話を見る導線）/ APA決済リンク承認 / レポート / アクティビティ

## 構成

- `index.html` — 全部入り（HTML/CSS/JS、外部依存ゼロ）
- `assets/avatar.png` — AI社員アバター

## コンセプト

- 人間の新入社員と同じく、AIも入社した瞬間に「育成」が必要
- ローカルファイル・コード・仕事の癖を100%理解させてから働かせる
- 理解が済めば A2A（エージェント同士の商談）と APA（決済はリンク承認だけ）が高品質になる
- エージェント同士の会話履歴は既存チャット（LINEグループ）で見る。ダッシュボードには成果物とステータスだけ
